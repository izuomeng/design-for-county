## 1. 产品定位

一个面向**县域商家**的轻量 AI 包装设计工具：商家通过对话式 Agent 提供必要信息，工具调用生图能力，生成可用于**自家产品包装落地制作**的包装初稿。

**一句话**：用 AI 帮"没钱、没设计、不懂设计"的县域商家，几分钟出一版能直接拿去打样制作的包装图。

> 说明：本工具产出的初稿，用途是商家自己的产品包装落地制作。与寻美的设计师比赛、作品库展示彼此独立，本文档不做关联设计。

---

## 2. 目标用户与场景

**用户画像**：县域农产品商家。典型特征——只确定知道"自己卖的是什么品类"，其余（风格、配色、文案、设计术语）都不懂；多用手机；没耐心、怕麻烦、不信任复杂工具。

**核心场景**：商家有一款产品（如土蜂蜜），想要一版像样、能打样制作的包装，但请不起设计师、也不会用专业设计软件。

**成功状态**：商家全程几乎不需要"想"和"打字描述"，靠点选就能拿到一版**文字清晰、可直接打样**的包装图并下载。

---

## 3. 设计总原则

面向"只知道自己卖什么、其余都不懂"的用户，最友好的设计是：**永远让用户"指"，不要让用户"想"。**

1. **看图点选，不用术语**：风格、配色都以图片选项呈现，用户看图点选，不读设计名词。
2. **凶狠的默认值 + "帮我决定"**：每一步都有"帮我选 / 没有 / 对"的兜底，用户一路点也能走到出图。
3. **步数极少、进度可见**：3~4 步内完成，用户随时清楚还剩几步。
4. **先出东西，再优化**：不一次问全，尽快出图，再基于结果迭代。
5. **Agent 的价值是"翻译"**：把商家说不清的需求，转成结构化的生图 prompt；对话是手段，结构化采集才是目的。

## 4. 交互形态

**双栏布局**（参考 Claude Desktop 等 Agent 产品）：

```
┌──────────────────────────────────────────────┐
│  寻美 · AI包装小助手                            │
├────────────────────┬─────────────────────────┤
│   左：对话 Agent     │   右：画布                │
│   - Agent 引导提问   │   - 风格参考 / 实时预览    │
│   - 用户点选/打字    │   - 生成结果 / 结果操作    │
│   - 快捷选项卡       │                          │
│   [输入框]          │                          │
└────────────────────┴─────────────────────────┘
```

- 左侧承载引导式对话，右侧承载视觉（风格参考图、实时预览、生成结果、结果操作）。
- 手机端：右画布在下方堆叠（响应式）。

## 5. 交互链路

```
[1 选品类] 用户唯一确定知道的事（蜂蜜/茶/果干/米/坚果…）
   │
   ▼
[2 选风格] 立刻展示同品类风格参考图，看图点选 + "都行你帮我选"
   │        ← 此步用图片完成"风格采集"，用户无需懂术语
   ▼
[3 填信息] 产品名 + 产地 + 产品照
   │        · 产地：合作县可自动预填
   │        · 产品照：支持"拍照/相册/没有，AI补图"
   │        · 卖点：由 Agent 自动猜，用户确认或微改
   ▼
[4 确认信息 · 准备生图] 右画布展示信息摘要卡 + 明确的「开始生成」按钮
   │
   ▼
[5 生成] 调用生图 API，一次生成 2~3 张方案，展示在右画布
   │
   ▼
[6 结果交互] 选一张喜欢 / 都不喜欢(重新生成) / 提修改意见(更新) / 下载
   │
   ▼
[7 出口] 下载高清图，用于自家产品包装落地制作
```

## 6. 关键功能需求

### 6.1 步骤 1~3 · 信息采集

| 步骤 | 需求 |
|---|---|
| 选品类 | 首屏不放空白对话框，直接以品类选项卡开场；支持自由输入兜底。 |
| 选风格 | 展示同品类风格参考图（图片为主，文字仅作小标签），点图选 1 个；提供"都行，你帮我选"默认项。 |
| 填产品名 | 提供建议名一键填入；支持自由输入。 |
| 填产地 | 合作县可自动识别预填，用户确认或改。 |
| 产品照 | 支持拍照、相册；提供"没有，AI 帮我画"兜底，无照片也能继续。 |
| 卖点 | Agent 根据品类自动猜测卖点，用户只需确认或微改，不强制商家写文案。 |

### 6.2 步骤 4 · 明确的生图动作

- Agent 收集完全部必要信息后，**不自动生成**，先进入"确认信息 · 准备生图"状态。
- 右画布展示信息摘要卡（风格 / 产品 / 产地 / 卖点）+ 一个醒目的「🎨 开始生成 · 一次出 2~3 张」主按钮。
- 点击该按钮 = 触发调用生图 API（明确的、用户主动发起的生图动作）。
- 生成过程展示 Loading 与分步状态文案（如"调用生图 API → 生成画面层 → 套清晰文字层"），约数秒返回。
- **一次性返回 2~3 张**方案并排展示在右画布。

### 6.3 步骤 6 · 生成后的结果交互

用户拿到 2~3 张结果后，可执行以下动作，覆盖所有去向，避免"出完图就没下文"：

| 动作 | 行为 |
|---|---|
| 选一张喜欢 | 点击某张 → 放大该方案，进入"下载/修改"页 |
| 都不喜欢 | 换一批风格，重新生成 2~3 张 |
| 提修改意见 | 用大白话输入（如"颜色淡一点""字再大点""换个背景"）或点快捷项 → Agent 理解后**更新**，重新生成 |
| 重新生成 | 同风格再出一版 |
| 下载 | 导出高清图，用于自家包装落地制作 |

- 修改意见同时支持"对话输入框打字"与"快捷选项点选"两种方式。
- 选中某张后的操作页提供：下载、还想改（提意见 → 更新）、重新生成、看回多张。

## 7. 关键技术要点

### 7.1 画面层 / 文字层分离（P0 必须实现）

扩散类生图模型生成中文文字几乎必然糊、错、乱排。若让 AI 直接"画出带产品名的包装"，产出的是看着像包装、但文字不可用的概念图，商家拿去打样会发现字全错。

```
  生图 ──▶ [画面层] 背景/风格/氛围/插画   ← AI 生成（擅长画面）
              │
              ▼ 叠加
        [文字层] 产品名/产地/卖点         ← 模板精确套字（清晰、可印刷）
              │
              ▼
        合成 = 真正可用、可打样的包装图
```

AI 只负责"美的画面"，所有汉字用模板精确套上。这是工具从"玩具"变成"能落地制作的工具"的分水岭，为 **P0** 必须项。

### 7.2 风格锚定

生成应约束在预设的"寻美风格库"内（沉淀自既有县域设计语言），避免产出通用 AI 味、风格漂移。实现方式可选：参考图引导、预设风格、或风格微调模型。

### 7.3 落地制作可用性

下载产物需满足打样制作的基本要求：足够分辨率、文字清晰可印刷。（出血/尺寸/格式规格待与制作方对齐后细化。）

### 7.4 内容安全

对生成内容做基础合规过滤，避免低质或违规图样在挂寻美/阿里公益品牌下产出。

## 8. 生图 API 接入（技术衔接点）

当前原型用本地 SVG 模拟生成结果。接入真实生图能力时，衔接点在"开始生成"触发的生成流程内：

```
点击「开始生成」
   → Agent 组装 prompt（结构化信息 + 寻美风格锚 + 品类版式约束）
   → 调用生图 API
   → 拿到 2~3 张画面层结果
   → 叠加文字层（模板套字）
   → 渲染到右画布
```

**Prompt 组装由 Agent 承担**：将商家点选/口述的信息，翻译成结构化、稳定的生图 prompt。

## 9. 风险与对策

| 风险 | 说明 | 对策 |
|---|---|---|
| 中文文字不可用 | 生图模型画中文字糊/错，打样无法用 | 画面层/文字层分离（P0） |
| 商家输入质量差 | 缺好图、缺卖点文案 | 凶狠默认值 + Agent 代猜卖点 + "AI 补图" |
| 风格漂移 | 产出通用 AI 味，不统一 | 风格锚定（参考图/预设/微调） |
| 纯聊天 UX 差 | 商家面对空对话框不知所措 | 引导式混合 UX（选项卡为主，自由输入兜底） |
| 落地制作不达标 | 分辨率/规格不符合打样 | 明确导出规格，与制作方对齐 |

---

## 10. 成功指标（建议）

| 指标 | 说明 |
|---|---|
| 工具进入数 | 进入工具的商家数 |
| **完成出图率** | 进入工具的商家最终成功拿到图的比例（**核心**，衡量对"什么都不懂"的用户是否友好） |
| 下载率 | 成功下载图的比例 |
| 修改/重生成使用率 | 反映结果交互被使用的程度与首版满意度 |
| 落地制作转化 | 下载后实际用于打样制作的比例（需外部回收） |
